# scripts/estimate — Manabi Map 独自偏差値推計（v1f 系）

Manabi Map の偏差値は商用偏差値サイトからの転載ではなく、**公的資料（県教育委員会の入試実施状況）と学校公式サイトの「入試結果」ページ**だけを入力にした独自推計です。このディレクトリはその推計ロジックの正典であり、透明性のために公開しています（データの実値はリポジトリに含めません）。

## 3 段階フロー

| 段階 | 条件 | 方法 |
|---|---|---|
| ① 式推計 | 学校公式「入試結果」等で合格最低点・受験者平均点・標準偏差が取れる | `偏差値 = 50 + 10 × (合格最低点 − 平均点) / 標準偏差`（四捨五入・30〜75 にクランプ） |
| ② 倍率参考推計 | ①が無く、志願者数と定員が取れる | 志願倍率 r = 志願者数 ÷ 定員。志願者集団を正規分布（平均 = baseline・既定 50）とみなし、合格ボーダーを下から (r−1)/r 分位点に置く: `偏差値 = baseline + 10 × Φ⁻¹((r−1)/r)`。r ≤ 1（定員充足）は選抜圧なしとみなしクランプ下限（38）を参考値とする（r→1+ の極限と連続）。信頼度が低いため 38〜66 にクランプし、note に「参考値」と明記。推計値の意味は①と同じく「合格ボーダー相当の難易度」 |
| ③ 情報募集中 | どちらも取れない | **数字を作らない**。SQL を出力せず「情報募集中」一覧（stderr）に載せる |

## 入力 CSV フォーマット

UTF-8・ヘッダ行必須・**1 ファイル = 1 県 × 1 年度**。実データ CSV は公開リポジトリに置かず `docs/local/`（gitignored）で管理する。

| 列 | 必須 | 内容 |
|---|---|---|
| `pref` | ○ | 県 slug（例: `tochigi`）。estimate_method の発番に使う |
| `year` | ○ | 対象年度（西暦・例: `2026`） |
| `school_name` | ○ | schools.name と完全一致する正式校名 |
| `department_name` | ○ | school_departments.name と完全一致する学科名 |
| `min_score` | 段階① | 合格最低点（学校公式の公表値のみ） |
| `exam_mean` | 段階① | 受験者平均点（県公表の入試実施状況等） |
| `exam_sd` | 段階① | 得点の標準偏差（県公表値。無ければ段階②へ） |
| `applicants` | 段階② | 志願者数 |
| `capacity` | 段階② | 募集定員 |
| `baseline_dev` | 任意 | 段階②の baseline 上書き（既定 50） |
| `source_url` | 推奨 | 出典 URL（公的資料・学校公式のみ。商用偏差値サイトの URL はエラーで拒否） |
| `note` | 任意 | 補足メモ |

段階の判定は行ごとに自動: ①の 3 値が揃えば①、揃わず②の 2 値が揃えば②、どちらも無ければ③。

## 使い方

```
node scripts/estimate/estimate-deviation.mjs --input <入力CSV> [--out <出力SQL>] [--pref <slug>] [--year <n>]

# 動作確認（合成データ・実在校ではない）
node scripts/estimate/estimate-deviation.mjs --input scripts/estimate/example-input.csv
```

- 出力は `school_deviation_values` への INSERT SQL（`begin;`/`commit;` 付き・校名/学科名のサブセレクトで id 解決）
- `estimate_method` は自動で `v1f_<pref>_<year>`（式推計系）。人手推計 `v1h_*` を本スクリプトが出すことはない
- `--pref` / `--year` は CSV 内容との一致検証用（省略可）
- 生成 SQL の本番適用は人間の目視確認 + 手動実行（AI は適用しない・CLAUDE.md 参照）

## estimate_method 命名規則（正典: docs/local/reference_school-data-collection-playbook.md）

- 式推計（本スクリプト）: `v1f_<県>_<年>`（種別を足す場合 `v1f_<県>_<種別>_<年>`・高校は種別省略）
- 人手主観推計（原則新規では使わない）: `v1h_<県>_<年>`
- 接尾辞なし `v1_<県>_<年>` は**廃止**（群馬 `v1_gunma_2026` は `v1h_gunma_2026` へ改名）
- ロジック改版時は `v2f_*` 等で新規発番し、旧レコードは `is_active=false`
- **県ごとに独立発番・独立算出。他県（群馬含む）の値の流用は禁止**

## provenance 原則（Non-negotiable）

- 商用偏差値サイト（みんこう・スタディサプリ進路・進学ネット等）の数値・URL を入力に混ぜない（スクリプトが domain 検査で拒否する）
- 段階②で数値が作れない校に、それらしい数字を手で埋めない（③「情報募集中」が正しい状態）
- 入力 CSV の実値・実在校サンプルをこのディレクトリにコミットしない（`example-input.csv` は合成データ）
